Attribute-Styles and Text Formatting

Without any CSS, we can set the font-family or the size or the c o l o r through <font face=... size=... color=...>.

We can give emphasis to some words by using the <em> tag or for even more power, try the <strong> tag!

How about something more esoteric, like the molecular formula for glucose C6H12O6 or the pythagorean theorem a2 + b2 + c2. You can use the <sup> or <sub> tags to denote superscript and subscript respectively.

Interactivity, Styling/Animation, and Going Forward

Unfortunately, we have reached just about the limits of what HTML alone is able to accomplish stylistically and interactively. Now, a few notes to the discerning programmer. This webpage was made as an extreme example of pushing HTML to its limits and being able to produce a reasonably tidy and responsive webpage without resorting to any CSS or JavaScript. However, there are some things that HTML (and this webpage) does well and others that it does not so well at and should never be done (but are done so in this case because of this unique constraint).

HTML, by its name, is a markup language which means that it is primarily an

  1. outline for labeling (describing) content
  2. instruction for how the content should be processed.
The key to understanding HTML is understanding its semantic and procedural nature.

It's semantic nature lies in its ability to

  • describe (through tags) what a block or section of content is (for example, <p>, <ul>, <figure>, <dl>, <img>, <audio>, <video>, <div>, etc.)
  • describe the semantic meaning of a certain word, line, or piece of text (whether it is <code>, <q>, <em>, <strong>, <a>, <abbr>, etc.)
  • how these blocks of content are grouped and sectioned in a general outline for the page (<header>, <footer>, <nav>, <section>, <article>, <h1-h6>).

Now the confusing part is how semantics and style are related to each other. You can clearly see on this page that there is some styling, not only on text, but also on the positioning of elements related to the page and each other. Some of these styles are applied automatically when we write an HTML tag, others are written ourselves through various HTML attributes and tricks regarding the table tag. There are exceptions, but a good general rule, is to never purposely try to apply any form of styles in the HTML document by yourself. This means that all of the styling attributes (size, color, bgcolor, align, face, border, width, height, background) are off limits. Using a table to position content is a definite no-no. In fact, a "correct" version of this page would look something along the lines of this.

Immediately we see that the general outline of the document is preserved. Upon further inspection of the source code, we see that HTML preserves what is most important: the content, the hierarchy, what each block of content is semantically meant to be (whether it is a paragraph, a table, a link, a heading section). Visually, we see the application of a few minimal styles (font size, emphasis, underline, italics) that help differentiate the semantic meaning of the tags.

This will allow us as programmers to separate content from style and content from interaction.

So how do we style or otherwise interact with elements?

This is where the procedural nature of HTML comes into play. HTML allows us to further classify each tag (piece of content) with a unique identifier (id) or group tags together (class). This allows us to target specific tags or groups of tags and apply a set of styles to those tags using style rules in a Cascading Style Sheet (CSS). We are also able to target these tags and make them interact with each other, with the browser environment, and programmatically control them through JavaScript (JS). A good way to remember this is that HTML is structural, CSS is presentational, and JavaScript is behavioral. While code for each should remain distinctly separate, they also interact with each other in many ways.

  • HTML help classify tags for identification within CSS and JS.
  • CSS can style HTML elements.
  • JS can programatically apply CSS styles.
  • JS can programatically modify the HTML document (DOM).
  • and many more ...

Hopefully this page was helpful in gaining a bit more insight into useful HTML tags, some creative applications, and how HTML is meant to be used.
