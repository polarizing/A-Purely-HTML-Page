Interactivity, Styling/Animation, and Going Forward

Unfortunately, we have reached just about the limits of what HTML alone is able to accomplish stylistically and interactively. Now, a few notes to the discerning programmer. This webpage was made as an extreme example of pushing HTML to its limits and being able to produce a reasonably tidy and responsive webpage without resorting to any CSS or JavaScript. However, there are some things that HTML (and this webpage) does well and others that it does not so well at and should never be done (but are done so in this case because of this unique constraint).

HTML, by its name, is a markup language which means that it is primarily an

  1. outline for labeling (describing) content
  2. instruction for how the content should be processed.
The key to understanding HTML is understanding its semantic and procedural nature.

It's semantic nature lies in its ability to

  • describe (through tags) what a block or section of content is (for example, a paragraph, list, figure, definition, image, audio, video, generic block etc.)
  • describe the semantic meaning of a certain word, line, or piece of text (whether it is code, a quote, emphasized text, strong text, hyperlink, abbreviation, etc.)
  • how these blocks of content are grouped and sectioned in a general outline for the page (header, footer, nav, section, article, h1-h6).

Now the confusing part is how semantics and style are related to each other. You can clearly see on this page that there is some styling, not only on text, but also on the positioning of elements related to the page and each other. Some of these styles are applied automatically when we write an HTML tag, others are written ourselves through various HTML attributes and clever tricks regarding the table tag. There are exceptions, but a good general rule, is to never purposely try to apply any form of styles in the HTML document by yourself. This means that all of the styling attributes (size, color, bgcolor, align, face, border, width, height, background) are off limits. Using a table to position content is a definite no-no.
